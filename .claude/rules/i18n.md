# Rule: Internationalization (i18n) — next-intl

Mọi chuỗi hiển thị cho người dùng PHẢI đi qua i18n. Nguồn chân lý là
`src/bootstrap/i18n/messages/{vi,en}.json` (decision `0020`). Không hardcode UI
copy trong `.tsx`/action.

## Single source + type-safe (cơ chế "const tổng")

- `vi.json` là **nguồn key duy nhất**; `en.json` PHẢI mirror đúng cấu trúc.
- Type augmentation ở `src/bootstrap/i18n/messages.d.ts`
  (`AppConfig.Messages = typeof vi.json`) → mọi `t("key")` được **check lúc
  compile**. Key sai = lỗi build = trace bug ngay. KHÔNG cần const literal riêng
  (sẽ drift khỏi JSON).
- Thêm key: sửa **cả** `vi.json` và `en.json` cùng lúc (cùng path).

## Nơi dịch (boundary) — dịch ở PRESENTATION

- Client Component: `useTranslations("<namespace>")`.
- Server Component (RSC/page): `await getTranslations("<namespace>")`.
- **Server Action / use-case / repository / domain KHÔNG dịch.** Trả về **key
  ổn định** (thường là `Failure["type"]`), presentation mới dịch.
  - Ví dụ: `loginAction` trả `{ errorKey: AuthFailure["type"] }`; key trùng path
    `auth.errors.<type>` → `tErrors(errorKey)` ở form.
- Lợi: tách i18n khỏi `'use server'`; failure union = catalog lỗi check
  end-to-end; khớp convention `attendance`.

## Namespace

- Theo feature/khu vực: `auth`, `attendance`, `shell`, `Common`, …
- Key dạng path lồng nhau (`auth.login.title`). Hyphen trong key OK
  (`invalid-credentials`); chỉ `.` là dấu phân tách namespace.
- Dynamic key chỉ dùng khi giá trị thuộc **union literal** đã có trong messages
  (vd `errorKey`, `labelKey`) — type vẫn check. KHÔNG truyền `string` thô vào
  `t()` (sẽ fail type — đó là chủ đích).

## KHÔNG phải i18n (giữ nguyên)

- Code comment, log, tên biến.
- **Mock/seed data** (tên người, môn học trong `*.mock.*`/`fixtures.ts`) là
  *dữ liệu*, không phải copy giao diện — không đưa vào messages.
- Proper noun thương hiệu (`EduPortal`).

## Kiểm chứng

- `bunx tsc --noEmit` bắt key sai / thiếu.
- Quét hardcoded (diacritics tiếng Việt) trong `.tsx`/`.ts` (trừ messages,
  `*.test.*`, `*.stories.*`, mock/fixtures) phải sạch.
- Story chạm UI vẫn qua design-review gate (`docs/DESIGN_REVIEW.md`).
